# Contributing to PluginWizard

We're happy to you're willing to contribute to PluginWizard.
We are not very strict about contributions, but we do have a few guidelines to help keep things organized.

If you don't know where to start, check out our [issues page](https://github.com/PluginWizard/PluginWizard/issues).

## Naming Conventions

When naming branches, please use the following conventions:
- `feature/your-feature-name` for new features
- `bugfix/your-bugfix-name` for bug fixes
- `docs/your-docs-update` for documentation updates
- `refactor/your-refactor-name` for code refactoring

## Pull Requests

When creating a pull request, please ensure that:
- Your code follows the existing coding style.
- You have updated documentation if necessary. (Developer or User docs)
- Your PR description clearly explains the changes made and the reason for them.
